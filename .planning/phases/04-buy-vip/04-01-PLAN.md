# PayTR callback (RESEARCH.md)
- Callback receives POST with merchant_oid, status, total_amount, hash (application/x-www-form-urlencoded).
- Verify hash: message = merchant_oid + merchant_salt + status + total_amount; expected = base64(HMAC-SHA256(merchant_key, message)); compare to post.hash.
- Response must be exactly "OK", Content-Type: text/plain; no HTML/JSON.
- Idempotency: before inserting VIP, check if a row with this merchant_oid already exists; if yes return OK without re-inserting.
- Do not send merchant_key or merchant_salt in get-token POST body (04-03 will use buildPaytrToken from this plan).
</context>
